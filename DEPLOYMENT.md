# Deployment Checklist & Quick Start Guide

This file documents what's included in the repository and what needs to be done after cloning.

## What's Included in the Repository

✅ **Source Code:**
- All PHP controllers, models, and routes
- All Blade template views
- Configuration files
- Database migrations
- Composer dependencies list (composer.json & composer.lock)
- NPM dependencies list (package.json & package-lock.json)

✅ **Configuration:**
- `.env.example` - Template for environment variables
- `.gitignore` - Excludes vendor/, node_modules/, .env, database files
- `.gitattributes` - Proper line ending handling
- `README.md` - Full setup and feature documentation

❌ **NOT Included (excluded by .gitignore):**
- `/vendor` - PHP dependencies (generated by composer install)
- `/node_modules` - Node dependencies (generated by npm install)
- `/.env` - Environment variables with sensitive data
- `/database/database.sqlite` - Local database (generated by php artisan migrate)
- `/storage/logs` - Log files
- `/storage/framework/cache` - Cache files
- `/bootstrap/cache` - Bootstrap cache files

## Quick Start After Cloning

### Step 1: Install Dependencies
```bash
composer install
npm install
```

### Step 2: Setup Environment
```bash
cp .env.example .env
php artisan key:generate
```

### Step 3: Setup Database
```bash
# SQLite (automatically used, no additional setup needed)
php artisan migrate

# OR for MySQL, update .env first with your credentials, then run migrate
```

### Step 4: Build Assets
```bash
npm run build
```

### Step 5: Start Development Server
```bash
php artisan serve
# Visit http://localhost:8000
```

## Database Info

- **Default**: SQLite (configured in .env, database file at `database/database.sqlite`)
- **Alternative**: MySQL (update DB_* variables in .env)
- **Migrations**: All tables created automatically via `php artisan migrate`

## For Production Deployment

1. Set `APP_ENV=production` in .env
2. Set `APP_DEBUG=false` in .env
3. Use MySQL database instead of SQLite
4. Run `php artisan migrate --force`
5. Run `npm run build` for optimized assets
6. Configure proper server (Nginx/Apache)
7. Set proper file permissions

## Troubleshooting

- **PHP extensions missing**: Enable pdo_mysql and pdo_sqlite in php.ini
- **Database errors**: Run `php artisan migrate` after .env setup
- **Assets not loading**: Run `npm run build`
- **Permission issues**: Check folder write permissions for storage/ and bootstrap/cache
